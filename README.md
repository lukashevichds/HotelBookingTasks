HotelBookingTasks


ТЕХНИЧЕСКОЕ ЗАДАНИЕ
на разработку программного приложения«Автоматизация решения задачи онлайн-бронирования и информационной поддержки клиентов с использованием технологий чат-ботов»
1. Цель разработки
Целью разработки является создание программного приложения, обеспечивающего:
	хранение и обработку данных о клиентах, отелях, бронированиях и задачах; 
	автоматическое формирование связанных задач по событиям бронирования; 
	контроль выполнения задач и мониторинг состояния системы; 
	информационную поддержку клиентов через встроенный чат-бот на основе заранее заданных правил и сценариев общения. 
2. Назначение приложения
Разрабатываемое приложение предназначено для автоматизации сопровождения клиентов сервиса онлайн-бронирования отелей. Система должна обеспечивать не только учет бронирований, но и формирование чек-листа действий, связанных с каждым бронированием, а также предоставление пользователю справочной информации через чат-бот.
3. Технологии для разработки приложения
Для реализации проекта будут использоваться следующие программные средства:
	язык программирования C#; 
	технология WPF; 
	платформа .NET; 
	PostgreSQL; 
	Visual Studio; 
	pgAdmin 4.

4. Архитектура приложения
Программное решение будет реализовано в виде десктопного приложения со следующей структурой:
	уровень представления — WPF-интерфейс; 
	уровень прикладной логики — классы и сервисы на языке C#; 
	уровень доступа к данным — модуль для выполнения SQL-запросов; 
	уровень хранения данных — база данных PostgreSQL; 
	модуль чат-бота — отдельный функциональный блок внутри логики приложения. 
5. Функциональные возможности приложения
5.1. Функции работы с пользователями
Приложение должно обеспечивать:
	авторизацию пользователя; 
	хранение учетных записей; 
	разграничение ролей пользователя и администратора; 
	отображение данных, доступных в рамках роли. 
5.2. Функции работы с бронированиями
Приложение должно обеспечивать:
	просмотр списка бронирований; 
	открытие карточки конкретного бронирования; 
	отображение параметров бронирования; 
	хранение связи между клиентом, отелем и бронированием; 
	возможность сопровождения бронирования через связанные задачи. 
5.3. Функции работы с задачами
Приложение должно обеспечивать:
	автоматическое формирование задач по выбранному бронированию; 
	отображение списка задач; 
	изменение статуса задачи; 
	контроль сроков выполнения; 
	фиксацию просроченных задач; 
	просмотр связанных задач в карточке бронирования. 
5.4. Функции мониторинга
Приложение должно обеспечивать:
	отображение статуса подключения к базе данных; 
	вывод количества бронирований; 
	вывод количества задач; 
	вывод количества просроченных задач; 
	отображение времени последнего обновления; 
	просмотр списка просроченных задач. 
5.5. Функции чат-бота
Приложение будет дополнено встроенным модулем чат-бота, который выполняет функцию информационной поддержки клиента.
Чат-бот должен:
	принимать текстовые обращения пользователя; 
	выполнять поиск ключевых слов и фраз; 
	определять тип запроса по заранее заданным шаблонам; 
	возвращать подготовленный ответ из набора сценариев; 
	при необходимости предлагать пользователю переход к нужному разделу системы; 
	обрабатывать типовые обращения, связанные с бронированием, оплатой, датами заезда, отменой, задачами и общей справкой по использованию системы. 
Чат-бот не должен использовать нейросетевую генерацию текста или внешние LLM-сервисы. Логика его работы должна строиться на правилах сопоставления пользовательского запроса со словарем ключевых фраз и заранее подготовленными ответами. Модуль чат-бота должен быть реализован как простой сценарный механизм с набором правил.



6. База данных
База данных PostgreSQL должна обеспечивать хранение следующих основных сущностей:
	пользователи; 
	отели; 
	бронирования; 
	задачи. 
7. Пользовательский интерфейс
Интерфейс приложения должен быть реализован средствами WPF и включать:
	окно авторизации; 
	главное окно; 
	страницу бронирований; 
	страницу задач; 
	страницу мониторинга; 
	карточку бронирования; 
	окно или панель чат-бота. 
Интерфейс должен быть логически понятным, единообразным по оформлению и обеспечивать быстрый переход между основными разделами приложения.
Для чат-бота должен быть предусмотрен отдельный визуальный элемент, включающий:
	поле отображения сообщений; 
	поле ввода текста; 
	кнопку отправки сообщения; 
	область вывода ответа. 
8. Надежность и сопровождение
Приложение должно обеспечивать:
	корректную работу при многократном выполнении основных операций; 
	устойчивость к ошибкам ввода пользователя; 
	контроль подключения к базе данных; 
	возможность повторного запуска без потери структуры данных; 
	удобство сопровождения кода за счет модульной структуры проекта. 
При отсутствии подключения к базе данных система должна корректно информировать пользователя о возникшей проблеме.
 
9. Тестирование
В рамках практической части будет проведено тестирование:
	авторизации; 
	загрузки списка бронирований; 
	открытия карточки бронирования; 
	автоматического формирования задач; 
	изменения статусов задач; 
	отображения мониторинговых показателей; 
	обработки запросов чат-ботом; 
	реакции чат-бота на неизвестные команды. 
10. Результаты разработки
По итогам выполнения практической части должны быть подготовлены:
	WPF-приложение; 
	база данных PostgreSQL; 
	модуль чат-бота на основе правил и ключевых фраз; 
	скриншоты работы приложения; 
	описание структуры системы; 
	описание логики работы чат-бота; 
	результаты тестирования; 
	эксплуатационная документация; 
	текст практической части дипломной работы. 
Результатом выполнения технического задания должна стать информационная система, которая:
	автоматизирует сопровождение клиентов сервиса онлайн-бронирования отелей; 
	формирует и отображает связанные задачи; 
	контролирует сроки их выполнения; 
	предоставляет мониторинг состояния системы; 
	обеспечивает базовую информационную поддержку пользователя через встроенный чат-бот.
